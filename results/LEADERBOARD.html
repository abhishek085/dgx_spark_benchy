Real numbers from actually running each model on one NVIDIA DGX Spark (128GB unified memory) — not vendor benchmarks. Every row below sent real requests to a real server and measured what happened.

generated from results/spark_bench_plus.csv · 6 model(s) benchmarked · sorted by completeness, then score, then most-recently tested

Status — Complete = ran the full eval+capacity+hermes+speed+checks pipeline. Partial = an older/legacy run, only has a subset of numbers; full re-test is queued (see the Model Wiki). Format — how compressed the model is (smaller = less memory, usually a little less accurate). LocalScore — 0-100 general quality score (accuracy + reliability + efficiency + responsiveness across 22 graded tasks). Independent of Hermes Score — see below. Hermes Score — a different 0-100 number specifically for "how good behind a Hermes-style personal agent": 50% real agent-task completion, 30% concurrent-session capacity, 20% responsiveness. A model can score well on one and poorly on the other (e.g. good general quality, but no tool-call parser wired up yet — see Model Wiki). Max Document Size — the largest amount of text we actually tested it with and got a real answer back (see the note above the table — this is the tested range, not necessarily the model's true breaking point). Peak Speed — decode tokens/sec at its best (smallest-context single-stream request). Max Concurrent Users — how many people/sessions could use it at the same time on this one box before it slowed down or started erroring. Tool Calling — whether it can reliably call functions/tools (needed for anything agentic, like web search or running code).
